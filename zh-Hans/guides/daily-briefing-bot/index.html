教程：构建每日简报机器人

在本教程中，您将构建一个个人简报机器人，它每天早上醒来，研究您关心的主题，总结发现，并将简洁的简报直接发送到您的 Telegram 或 Discord。

到最后，您将拥有一个完全自动化的工作流，结合了网络搜索、cron 调度、委托和消息传递 — 无需编写代码。

我们要构建什么​

以下是流程：

  1. 上午 8:00 — cron 调度器触发您的作业
  2. Hermes 启动一个带有您提示的新智能体会话
  3. 网络搜索获取您主题的最新新闻
  4. 摘要将其提炼成干净的简报格式
  5. 交付将简报发送到您的 Telegram 或 Discord

整个过程无需手动操作。您只需在早晨喝咖啡时阅读您的简报。

先决条件​

在开始之前，请确保您有：

  • Hermes Agent 已安装 — 请参阅安装指南
  • 网关正在运行 — 网关守护进程处理 cron 执行： 
    hermes gateway install   # 安装为用户服务
    sudo hermes gateway install --system   # Linux 服务器：启动时系统服务
    # 或
    hermes gateway           # 在前台运行
  • Firecrawl API 密钥 — 在您的环境中设置 FIRECRAWL_API_KEY 用于网络搜索
  • 消息传递已配置（可选但推荐）— 设置 Telegram 或 Discord 并配置一个家庭频道
没有消息传递？没问题

您仍然可以使用 deliver: "local" 来跟随本教程。简报将保存到 ~/.hermes/cron/output/，您可以随时阅读它们。

步骤 1：手动测试工作流​

在自动化任何内容之前，让我们确保简报工作正常。启动一个聊天会话：

hermes

然后输入此提示：

搜索关于 AI 智能体和开源 LLM 的最新新闻。
用简洁的简报格式总结前 3 个故事，并附带链接。

Hermes 将搜索网络，阅读结果，并生成类似以下内容：

☀️ 您的 AI 简报 — 2026 年 3 月 8 日

1. Qwen 3 发布，拥有 2350 亿参数
   阿里巴巴最新的开放权重模型在多个基准测试中与 GPT-4.5 匹配，
   同时保持完全开源。
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain 推出智能体协议标准
   一个新的智能体间通信开放标准在第一周就获得了 15 个主要框架的采用。
   → https://blog.langchain.dev/agent-protocol/

3. 欧盟 AI 法案对通用模型的执法开始
   第一个合规截止日期到来，低于 1000 万参数阈值的开源模型获得豁免。
   → https://artificialintelligenceact.eu/updates/

---
3 个故事 • 搜索来源：8 • 由 Hermes Agent 生成

如果这有效，您就可以自动化它了。

迭代格式

尝试不同的提示，直到获得您喜欢的输出。添加诸如"使用表情符号标题"或"将每个摘要保持在 2 句话以内"之类的说明。无论您决定什么，都会进入 cron 作业。

步骤 2：创建 Cron 作业​

现在让我们安排它每天早上自动运行。您可以通过两种方式完成此操作。

选项 A：自然语言（在聊天中）​

只需告诉 Hermes 您想要什么：

每天早上 8 点，搜索网络以获取关于 AI 智能体和开源 LLM 的最新新闻。
用简洁的简报总结前 3 个故事，并附带链接。使用友好、专业的语气。交付到 telegram。

Hermes 将使用统一的 cronjob 工具为您创建 cron 作业。

选项 B：CLI 斜杠命令​

使用 /cron 命令进行更多控制：

/cron add "0 8 * * *" "搜索网络以获取关于 AI 智能体和开源 LLM 的最新新闻。从过去 24 小时中找到至少 5 篇最新文章。用简洁的每日简报格式总结前 3 个最重要的故事。对于每个故事，包括：清晰的标题、2 句话摘要和来源 URL。使用友好、专业的语气。使用表情符号项目符号格式，并以故事总数结束。"

黄金法则：自包含提示​

关键概念

Cron 作业在完全新鲜的会话中运行 — 没有以前对话的记忆，没有关于您"之前设置"的上下文。您的提示必须包含智能体完成工作所需的一切。

糟糕的提示：

做我通常的早间简报。

好的提示：

搜索网络以获取关于 AI 智能体和开源 LLM 的最新新闻。
从过去 24 小时中找到至少 5 篇最新文章。用简洁的每日简报格式
总结前 3 个最重要的故事。对于每个故事，包括：清晰的标题、
2 句话摘要和来源 URL。使用友好、专业的语气。使用表情符号项目符号格式。

好的提示具体说明了搜索什么、多少篇文章、什么格式和什么语气。这是智能体一次性所需的一切。

步骤 3：自定义简报​

一旦基本简报工作正常，您就可以发挥创意了。

多主题简报​

在一个简报中涵盖多个领域：

/cron add "0 8 * * *" "创建一个涵盖三个主题的早间简报。对于每个主题，搜索网络以获取过去 24 小时的最新新闻，并用链接总结前 2 个故事。

主题：
1. AI 和机器学习 — 专注于开源模型和智能体框架
2. 加密货币 — 专注于比特币、以太坊和监管新闻
3. 太空探索 — 专注于 SpaceX、NASA 和商业太空

格式为带有章节标题和表情符号的干净简报。以今天的日期和一句励志名言结束。"

使用委托进行并行研究​

为了更快的简报，告诉 Hermes 将每个主题委托给子智能体：

/cron add "0 8 * * *" "通过将研究委托给子智能体来创建早间简报。委托三个并行任务：

1. 委托：搜索过去 24 小时内带有链接的前 2 个 AI/ML 新闻故事
2. 委托：搜索过去 24 小时内带有链接的前 2 个加密货币新闻故事
3. 委托：搜索过去 24 小时内带有链接的前 2 个太空探索新闻故事

收集所有结果并将它们组合成一个带有章节标题、表情符号格式和来源链接的单一干净简报。将今天的日期添加为标题。"

每个子智能体独立并行搜索，然后主智能体将所有内容组合成一个经过润色的简报。有关其工作原理的更多信息，请参阅委托文档。

仅工作日计划​

周末不需要简报？使用针对周一至周五的 cron 表达式：

/cron add "0 8 * * 1-5" "搜索最新的 AI 和科技新闻..."

每日两次简报​

获取早间概览和晚间回顾：

/cron add "0 8 * * *" "早间简报：搜索过去 12 小时的 AI 新闻..."
/cron add "0 18 * * *" "晚间回顾：搜索过去 12 小时的 AI 新闻..."

使用记忆添加个人上下文​

如果您启用了记忆，您可以存储跨会话持久存在的偏好。但请记住 — cron 作业在没有对话记忆的新鲜会话中运行。要添加个人上下文，请直接将其烘焙到提示中：

/cron add "0 8 * * *" "您正在为一位关心以下内容的高级 ML 工程师创建简报：PyTorch 生态系统、transformer 架构、开放权重模型和欧盟的 AI 监管。除非涉及开源，否则跳过关于产品发布或融资轮次的故事。

搜索这些主题的最新新闻。用链接总结前 3 个故事。简洁且技术性强 — 这位读者不需要基本解释。"
定制角色

包含关于简报是为谁的详细信息会显著提高相关性。告诉智能体您的角色、兴趣和要跳过的内容。

步骤 4：管理您的作业​

列出所有计划作业​

在聊天中：

/cron list

或从终端：

hermes cron list

您将看到类似以下的输出：

ID          | 名称              | 计划    | 下次运行           | 交付
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | 早间简报  | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | 晚间回顾     | 0 18 * * *  | 2026-03-08 18:00   | telegram

删除作业​

在聊天中：

/cron remove a1b2c3d4

或通过对话询问：

删除我的早间简报 cron 作业。

Hermes 将使用 cronjob(action="list") 查找它，并使用 cronjob(action="remove") 删除它。

检查网关状态​

确保调度器实际正在运行：

hermes cron status

如果网关未运行，您的作业将不会执行。将其安装为后台服务以确保可靠性：

hermes gateway install
# 或在 Linux 服务器上
sudo hermes gateway install --system

进一步探索​

您已经构建了一个可用的每日简报机器人。以下是一些接下来可以探索的方向：

  • 计划任务 (Cron) — 计划格式、重复限制和交付选项的完整参考
  • 委托 — 深入探讨并行子智能体工作流
  • 消息平台 — 设置 Telegram、Discord 或其他交付目标
  • 记忆 — 跨会话的持久上下文
  • 提示与最佳实践 — 更多提示工程建议
您还可以计划什么？

简报机器人模式适用于任何内容：竞争对手监控、GitHub 仓库摘要、天气预报、投资组合跟踪、服务器健康检查，甚至是每日笑话。如果您可以在提示中描述它，就可以安排它。
