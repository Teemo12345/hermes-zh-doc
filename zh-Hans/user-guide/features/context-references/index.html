上下文引用

输入 @ 后跟引用，将内容直接注入到您的消息中。Hermes 会内联展开引用，并将内容附加到 --- Attached Context --- 部分下。

支持的引用​

语法描述
@file:path/to/file.py注入文件内容
@file:path/to/file.py:10-25注入特定行范围（从 1 开始索引，包含两端）
@folder:path/to/dir注入带有文件元数据的目录树列表
@diff注入 git diff（未暂存的工作树更改）
@staged注入 git diff --staged（已暂存的更改）
@git:5注入最后 N 个带有补丁的提交（最多 10 个）
@url:https://example.com获取并注入网页内容

使用示例​

Review @file:src/main.py and suggest improvements

What changed? @diff

Compare @file:old_config.yaml and @file:new_config.yaml

What's in @folder:src/components?

Summarize this article @url:https://arxiv.org/abs/2301.00001

单个消息中可以使用多个引用：

Check @file:main.py, and also @file:test.py.

尾随标点符号（,, ., ;, !, ?）会自动从引用值中剥离。

CLI 标签补全​

在交互式 CLI 中，输入 @ 会触发自动补全：

  • @ 显示所有引用类型（@diff, @staged, @file:, @folder:, @git:, @url:）
  • @file: 和 @folder: 触发带有文件大小元数据的文件系统路径补全
  • 裸 @ 后跟部分文本显示当前目录中匹配的文件和文件夹

行范围​

@file: 引用支持行范围以进行精确的内容注入：

@file:src/main.py:42        # 单行 42
@file:src/main.py:10-25     # 从第 10 行到第 25 行（包含两端）

行是从 1 开始索引的。无效范围会被静默忽略（返回完整文件）。

大小限制​

上下文引用有限制，以防止超出模型的上下文窗口：

阈值值行为
软限制上下文长度的 25%附加警告，继续展开
硬限制上下文长度的 50%拒绝展开，返回未更改的原始消息
文件夹条目最多 200 个文件多余条目被替换为 - ...
Git 提交最多 10 个@git:N 被限制在 [1, 10] 范围内

安全性​

敏感路径阻止​

这些路径始终被 @file: 引用阻止，以防止凭据泄露：

  • SSH 密钥和配置：~/.ssh/id_rsa、~/.ssh/id_ed25519、~/.ssh/authorized_keys、~/.ssh/config
  • Shell 配置文件：~/.bashrc、~/.zshrc、~/.profile、~/.bash_profile、~/.zprofile
  • 凭证文件：~/.netrc、~/.pgpass、~/.npmrc、~/.pypirc
  • Hermes 环境：$HERMES_HOME/.env

这些目录完全被阻止（内部的任何文件）：

  • ~/.ssh/、~/.aws/、~/.gnupg/、~/.kube/、$HERMES_HOME/skills/.hub/

路径遍历保护​

所有路径都相对于工作目录解析。解析到允许的工作区根目录之外的引用会被拒绝。

二进制文件检测​

二进制文件通过 MIME 类型和空字节扫描进行检测。已知的文本扩展名（.py、.md、.json、.yaml、.toml、.js、.ts 等）绕过基于 MIME 的检测。二进制文件会被拒绝并显示警告。

平台可用性​

上下文引用主要是CLI 功能。它们在交互式 CLI 中工作，其中 @ 触发标签补全，并且引用在消息发送到智能体之前被展开。

在消息平台（Telegram、Discord 等）中，网关不会展开 @ 语法 — 消息会按原样传递。智能体本身仍然可以通过 read_file、search_files 和 web_extract 工具引用文件。

与上下文压缩的交互​

当对话上下文被压缩时，展开的引用内容会包含在压缩摘要中。这意味着：

  • 通过 @file: 注入的大文件内容会增加上下文使用量
  • 如果对话后来被压缩，文件内容会被总结（不会逐字保留）
  • 对于非常大的文件，考虑使用行范围（@file:main.py:100-200）只注入相关部分

常见模式​

# 代码审查工作流程
Review @diff and check for security issues

# 带上下文的调试
This test is failing. Here's the test @file:tests/test_auth.py
and the implementation @file:src/auth.py:50-80

# 项目探索
What does this project do? @folder:src @file:README.md

# 研究
Compare the approaches in @url:https://arxiv.org/abs/2301.00001
and @url:https://arxiv.org/abs/2301.00002

错误处理​

无效引用会产生内联警告而不是失败：

条件行为
文件未找到警告："file not found"
二进制文件警告："binary files are not supported"
文件夹未找到警告："folder not found"
Git 命令失败带有 git 标准错误的警告
URL 返回无内容警告："no content extracted"
敏感路径警告："path is a sensitive credential file"
路径在工作区外警告："path is outside the allowed workspace"
